網址縮短程序Polr搭建教程

發表於2021-10-09|更新於2021-10-10|技術帖
|字數總計:842|閱讀時長:2分鐘|閱讀量:

GitHub

cydrobolt/polr

功能介紹

  1. 修改縮短的域名
  2. 統計功能(來源，時間)
  3. API支持
  4. 二維碼生成

服務器要求

  • Apache，nginx，IIS或lighttpd
  • PHP> = 5.5.9
  • MariaDB或MySQL> = 5.5，或者使用SQLite
  • PHP要求：
    1. OpenSSL PHP擴展
    2. PDO PHP擴展
    3. PDO MySQL驅動程序（Debian和Ubuntu上的php5-mysql，FreeBSD上的php5x-pdo_mysql）
    4. Mbstring PHP擴展
    5. Tokenizer PHP擴展
    6. JSON PHP擴展
    7. PHP curl擴展

項目開源地址

https://phoenix.yizimg.com/cydrobolt/polr

此時搭建完，看似都部署完成了，按照網上教程大都到這裡就提示你們可以訪問自己網站，進入設定囉！但是事實上是這樣嗎？

訪問域名後發現一個嚴重錯誤

個人經歷，我跟大多數人一樣，按照教程搭建好就直接開心的訪問域名了，一開始正常的設定，一切很順利的按照提示綁定域名、綁定數據庫、一些相關設定，按下「安裝」按鈕後，卻出現了令人絕望的「Whoops, looks like something went wrong.」

為何會這樣？

查看github issue後，很多人都反映了這問題，但作者似乎已經不打算繼續維護項目，雖然有些大神找出辦法，但完全難以理解，講白了根本看不懂在幹嘛…..。

後來看到關鍵字，思索一番，決定翻查一下數據庫，我個人直覺認為數據庫一定出了問題。因為管理員帳號明顯進不去，代表根本沒有成功註冊！

果然不出所望，數據庫一片空白～

解決辦法

選擇相對應的數據庫名稱，點擊匯入重構的數據庫文件

下載重構的數據庫資料

下載完後按照上圖導入

之後你就可以正常訪問你搭建的短網址網站啦！
